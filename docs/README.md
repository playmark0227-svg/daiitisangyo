## このプロジェクトの要点（3ドキュメント共通の前提）

- **提供形態**: スマートフォンアプリで完結（Web ECではない）。初期はQRコード経由のクローズドβ → テスト後に一般公開の段階リリース。
- **3ロール**: 管理者（運営会社）／出品者（漁師・農家・漁協）／発注者（飲食店・百貨店・惣菜工場）。
- **最優先UX**: 高齢者・アナログ層の出品者でも使える極限までシンプルな操作。出品は「①写真を撮る→②金額（原価）を入れる→③出品ボタン」の3操作。
- **価格モデル（加算モデル）**: 出品者が原価を入力 → 管理者設定のマージンを加算 → 販売価格を自動算出し発注者へ総額表示（要件F-007）。
- **決済**: Stripe。売上分配を伴うためStripe Connectを推奨候補とするが、**取引スキーム（買取再販 or マーケットプレイス仲介）の確定が最優先**（資金決済法の論点に直結）。
- **納期**: 3〜4ヶ月でフェーズ1（コアループ：出品→閲覧→注文→決済→発送）を完成。

## フェーズ用語（3ドキュメント共通の定義）

- **フェーズ1**: 今回開発（3〜4ヶ月）。
- **リリース1.x**: フェーズ1完了後の早期機能拡充（キーワード検索、通知一覧タブ、出品者売上確認画面、LINE公式アカウント連携、送り状API連携、インボイス本格対応、ダッシュボード拡充 等）。
- **フェーズ2（事業拡大）**: 道の駅・温泉施設商品、市町村・自治体の横展開、数百億円規模GMVを見据えた拡張。

## 次回打ち合わせでの最優先確認事項（要件確定・開発着手前に必須）

要件定義書 第10章に全23項目を整理しています。特に開発着手前に回答が必要なもの:

1. **取引スキーム（買取再販 or マーケットプレイス仲介）と資金フロー** — 資金決済法（資金移動業登録の要否）に直結。**最優先**。（第10章#5）
2. **配送・受け渡し方法と送料** — 宅配か直接受け渡しか、送料の負担・算定方式、クール便対応。配送先住所入力の要否にも影響。（第10章#1）
3. **決済確定タイミング** — 即時決済 or オーソリ→キャプチャ（欠品・キャンセルの多い生鮮品では後者が有利）。（第10章#22）
4. **返品・キャンセルポリシー**（生鮮品特性）／**請求書払い（掛売り）の要否**（BtoB想定）／**インボイス・消費税（軽減税率）対応**。（第10章#2〜#4）
5. **アプリ実装形態**（ネイティブ/クロスプラットフォーム/PWA）と**通知方式**（アプリプッシュ先行／LINE連携）。（第10章#7・#8）
6. **発注者の属性**（事業者限定か一般消費者を含むか）— 特商法表記の要否等に波及。（第10章#9）

- 本ドキュメントの工数・費用・期間・数値はすべて概算（仮置き）です。詳細見積は要件確定後に別途提示します。
